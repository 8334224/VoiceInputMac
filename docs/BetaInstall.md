# Beta 安装与分发说明

## 当前最小分发方式

当前项目可以通过打包脚本生成：

- `.app`：`dist/VoiceInputMac.app`
- `.zip`：`dist/VoiceInputMac-0.1.0-macOS.zip`

在项目根目录执行：

```bash
./scripts/build_app.sh
```

脚本会：

- 生成 release 可执行文件
- 组装 `VoiceInputMac.app`
- 做一次 ad-hoc 签名
- 生成 zip 压缩包

## 非开发者如何安装

1. 获取 `VoiceInputMac-0.1.0-macOS.zip`
2. 双击解压
3. 把 `VoiceInputMac.app` 拖到 `Applications`
4. 首次打开时，如果 macOS 阻止启动：
   - 右键应用
   - 选择“打开”
   - 再确认一次

## 首次启动需要的权限

首次实际使用时，应用会依次请求：

- 麦克风
- 语音识别

如果启用了自动粘贴，还需要：

- 辅助功能

其中：

- 麦克风：用于采集语音
- 语音识别：用于 Apple Speech 首轮转写
- 辅助功能：用于把结果自动粘贴到当前输入框

## 首次启动建议顺序

1. 打开应用
2. 点击菜单栏麦克风图标
3. 先做一次短录音
4. 按系统弹窗授予：
   - 麦克风
   - 语音识别
5. 如果要自动粘贴，再去系统设置开启辅助功能

## 常见报错与处理

### 1. 权限不足

表现：

- 状态显示“权限不足”

处理：

- 去 `系统设置 > 隐私与安全性`
- 打开：
  - 麦克风
  - 语音识别

### 2. 麦克风不可用

表现：

- 状态显示“麦克风不可用”
- 或提示没有可用输入设备

处理：

- 检查是否接好了输入设备
- 检查 `系统设置 > 声音 > 输入`
- 关闭占用麦克风的应用后重试

### 3. 自动粘贴没生效

表现：

- 识别成功，但没有自动插入到当前输入框

处理：

- 去 `系统设置 > 隐私与安全性 > 辅助功能`
- 允许本应用
- 如果刚刚勾选，最好完全退出再重新打开应用

### 4. 快捷键无效

表现：

- 按快捷键没有反应

处理：

- 去设置页更换快捷键
- 如果快捷键被别的应用占用，应用会保留上一个可用快捷键
- 也可以直接点击菜单栏里的“开始听写”

## 当前签名 / notarization 状态

当前状态：

- 已做 ad-hoc 签名
- 还没有 Developer ID 签名
- 还没有 notarization

这意味着：

- 自己分发给 Beta 测试者是可以的
- 但首次打开时仍可能遇到 Gatekeeper 提示
- 还不适合做正式公开下载页分发

## Beta 前剩余发布阻塞点

至少还包括：

- Developer ID Application 签名
- notarization
- 更正式的版本号与 release note
- 一轮真实机器上的首次安装验证

更完整的正式发布准备见：

- [`docs/ReleaseSigning.md`](ReleaseSigning.md)

## 推荐的 Beta 分发方式

当前最现实的是：

1. 用 `./scripts/build_app.sh` 生成 zip
2. 把 zip 发给少量测试者
3. 附上本说明
4. 先收集：
   - 权限问题
   - 热键冲突
   - 自动粘贴失败
   - 首次启动被拦截的情况
